Quality: About This Data Set

The data was collected with the help of Aware Framework. The data primarily contains GPS sensor information over a week (October 09, 2018 to October 16, 2018) along with the timestamps. The table contains arrival timestamp, departure timestamp, address of the location, lat-long of the location and accuracy.

There were numerous issues with the collected data like timestamps being incorrect (too huge to be true or negative), etc.

Hence we had to check this data for Completeness, Correctness and Coherency. We perform exploratory analysis using Pandas in Python and some transformations using Trifacta Wrangler. The data was cleaned to make sure that it is reliable to answer the question we pose.

Is the data complete?

Yes, the data is complete as it was analyzed with the help of Pandas which reported that there were 6 groups of addresses after creating groups based on lat-long with accuracy percentage in Pandas. The 6 groups were manually verified to be complete by the subject himself. The 6 groups were Carnegie Mellon University, Craig Street, Morewood Avenue, Howe Street, Negle Avenue, Bellefonte Street. However, the dataset did not contain proper attributes to answer the question. For this, new derived attributes were created using the existing ones. For example, to calculate the duration Shaleen was at a particular location, it was derived by subtracting arrival timestamp from departure timestamp and converting units to minutes. Similarly, since GPS is usually more accurate than required, the latitudes and longitudes were grouped together in bins based on their accuracy of reporting to form groups. These groups represent individual buildings/places Shaleen visited, which is more useful as opposed to individual latitudes and longitudes. With these derived quantities, the dataset was complete enough to make relevant visualizations to answer the question.

Is the data coherent?

After the data's correctness was dealt with, the data was established to be coherent too. Now each attribute was having their values in the expected range and matched well with the dataset in general. Only the attributes which were relevant to the question was kept in the final table view that was used to make the visualizations. There was no redundancy after we removed the attributes which were used once to create a derived quantity like duration of time spent at a particular place. So was verified by importing data in Trifacta Wrangler as well.

Is the data correct?

There were some timestamps which were negative and some that were coming out to be too huge to be true (eg in the order of 10^30). There were 6 rows with such data representing only a very insignificant portion of the entire dataset. Hence, these 6 rows were safely dropped with the help of Trifacta Wrangler to make the data clean and correct. After this step, it was established with the help of Trifacta Wrangler that the dataset was correct. Wrangler was able to identify the data type of each field with no mismatches. We concluded in the end that the dataset was no correct.

Accountability:

The dataset was obtained by collecting sensor information from the subject's mobile phone with the help of Aware Framework. The subject could verify that the phone was within a range of 10ft from him for the entire duration of the study and the he could cross-verify the final dataset used for answering the question, it is fair game to establish that the data is accountable. However, there is a scope of error in the data collection by the Aware framework (which is known to be a ell tested framework) and inaccuracies due to misconfigured/broken sensors in the phone. We also assume that the subject is truthful and has enough memory of last week to testify that the data is correct, complete and coherent. As long as the subject, their phone and the aware framework are accountable, the dataset is accountable.
